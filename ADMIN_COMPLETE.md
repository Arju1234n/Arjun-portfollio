## ⚠️ Configuration Notes

### Required for Production

#### 1. Cloudinary (Image Uploads)
```env
# backend/.env
CLOUDINARY_CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=your-api-key
CLOUDINARY_API_SECRET=your-api-secret
```

#### 2. Resend (Email Service)
```env
# backend/.env
RESEND_API_KEY=re_your-api-key
EMAIL_FROM=Portfolio <noreply@yourdomain.com>
ADMIN_EMAIL_NOTIFY=your@email.com
```

#### 3. JWT Secrets (Change in Production!)
```env
# backend/.env
JWT_ACCESS_SECRET=generate-strong-secret-here
JWT_REFRESH_SECRET=generate-another-strong-secret
```

### Optional Configurations

#### Rate Limiting
Currently set to 1000 requests per 15 minutes for development.  
For production, adjust:
```env
RATE_LIMIT_MAX=100  # Stricter for production
```

#### CORS Origins
Update for production domains:
```env
CORS_ORIGINS=https://yourdomain.com,https://admin.yourdomain.com
```

### Backend
**Recommended**: Railway, Render, or Vercel
- Set environment variables
- Update MongoDB IP whitelist
- Configure domain/CORS
- Enable production mode

### Frontend  
**Recommended**: Vercel or Netlify
- Connect GitHub repo
- Set `NEXT_PUBLIC_API_URL`
- Configure build settings
- Enable automatic deployments

### Admin Dashboard
**Recommended**: Vercel (separate deployment)
- Use different subdomain (admin.yourdomain.com)
- Set `NEXT_PUBLIC_API_URL`
- Restrict access via auth
- Consider IP whitelisting
